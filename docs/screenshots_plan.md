# Screenshots Plan

This repository is evidence-first, so screenshots should support navigation and
credibility without replacing the original certificate PDFs.

## Goals

- Show breadth across Bitrix24, 1C, BPMN/eEPC, and analyst tracks.
- Keep screenshots simple enough for GitHub profile, CV, and portfolio usage.
- Avoid heavy image maintenance where a PDF or README link is stronger.

## Recommended Screens

1. Repository root with category structure visible.
2. Bitrix24 folder view showing administration / automation / CRM tracks.
3. One category README opened as an example navigation page.
4. Optional CV PDF preview thumbnail for portfolio materials.

## Capture Rules

- Do not crop out filenames or dates that help verify authenticity.
- Avoid duplicating certificate page content when a PDF link is clearer.
- Keep screenshots focused on repository structure and reviewability.

## Packaging Targets

- GitHub profile references
- CV / portfolio attachments
- LinkedIn / Notion portfolio pages
